# 歌单海报 — 分类知识库

## 定义
歌单海报是用于分享音乐歌单的视觉设计作品，通常在社交媒体（朋友圈、小红书、微博等）传播。目标是让观众快速理解歌单主题、氛围，并产生收听欲望。

## 核心元素（必须出现）

| 元素 | 说明 | 视觉特征 |
|------|------|---------|
| 歌单名称/标题 | 歌单的核心主题表达 | 通常是最醒目的文字，字号最大，有装饰效果 |
| 歌曲列表 | 歌单中的歌曲名称 | 多列或单列排列，字号较小，有序号或装饰符号 |
| 标题卡片 | 标题文字的视觉容器 | 可有圆角、描边、发光、装饰底纹，但不包含文字 |
| 歌单卡片 | 歌曲列表的视觉容器 | 可有圆角、边框、分栏线、装饰底纹，但不包含歌曲名 |

## 常见元素（经常出现，非必须）

| 元素 | 说明 | 视觉特征 |
|------|------|---------|
| 主视觉/插画 | 海报的核心视觉焦点，可以是人物、场景、抽象图形 | 占据较大面积，风格化处理 |
| 背景 | 整体氛围基底 | 可以是纯色、渐变、纹理、场景插画 |
| 装饰元素 | 花朵、星星、音符、贴纸、边框等 | 点缀性，增强氛围 |
| 副标题/描述 | 歌单的补充说明文字（如"适合深夜独处时听"） | 字号中等，辅助性 |
| 作者/策划者 | 歌单创建者信息 | 字号小，通常在角落 |
| 播放按钮/音乐图标 | 暗示音乐属性的视觉符号 | 小面积装饰性元素 |
| 歌曲数量 | 如"32首" | 小字标注 |
| 音乐平台标识 | 如 Spotify、网易云音乐 logo | 极小，角落放置 |

## 不常见的元素（特定风格才会出现）

| 元素 | 说明 | 何时出现 |
|------|------|---------|
| 音乐播放器 UI | 模拟真实播放器界面的装饰 | 复古/拟物风格 |
| 人物角色插画 | 与歌单主题相关的人物 | 二次元/插画风格 |
| 歌词引用 | 某首歌的经典歌词 | 文艺/情绪风格 |
| 二维码 | 链接到实际播放列表 | 功能性海报 |

## 常见布局模式

### 模式 A：标题居上 + 主视觉居中 + 歌单居下
```
┌─────────────────────┐
│     歌单标题         │  ← 大字标题，装饰性
│                     │
│    ┌───────────┐    │
│    │  主视觉    │    │  ← 插画/照片/图形
│    │  (人物/场景)│    │
│    └───────────┘    │
│                     │
│  歌曲1  歌曲5  歌曲9 │  ← 多列歌单
│  歌曲2  歌曲6  歌曲10│
│  歌曲3  歌曲7       │
│  歌曲4  歌曲8       │
└─────────────────────┘
```

```
┌─────────────────────┐
│ 标题   ┌────┐       │

```

## 风格光谱

| 风格方向 | 色调 | 典型元素 | 适合歌单类型 |
|---------|------|---------|-------------|
| 梦幻少女 | 粉色、浅紫、奶油色 | 星星、爱心、蝴蝶结、花朵 | 甜系/治愈系歌单 |
| 暗黑酷感 | 黑、深紫、霓虹 | 几何图形、闪电、金属质感 | 摇滚/电子/说唱歌单 |
| 复古怀旧 | 暖黄、棕色、做旧质感 | 胶片纹理、老式收音机、手写字 | 老歌/民谣歌单 |
| 清新自然 | 绿色、天蓝、白色 | 植物、阳光、云朵、手绘 | 轻音乐/独立歌单 |
| 赛博未来 | 深蓝、霓虹粉、荧光绿 | 光线、网格、故障效果 | 电子/合成器歌单 |
| 极简现代 | 黑白灰 + 一个强调色 | 干净排版、大留白、无衬线字体 | 通用/品味展示 |
| 中式国风 | 中国红、墨色、金色 | 水墨、书法、古典纹样 | 古风/国风歌单 |

## 文本样式规律

| 文字角色 | 典型字号（相对390宽画布） | 典型处理 |
|---------|----------------------|---------|
| 歌单标题 | 24-36px | 粗体，有描边/发光/阴影等装饰效果 |
| 副标题/描述 | 12-16px | 常规体，辅助色 |
| 歌曲名 | 8-12px | 多列排列，有序号装饰 |
| 作者/署名 | 8-10px | 低调，角落位置 |

## 生成约束参考

### 默认最小组件集合
用户没有补充说明时，只生成以下组件：
- 背景（图片）
- 标题（文字，contentSource = user.title）
- 歌单（文字，contentSource = user.songs）
- 标题卡片（图片，只是标题容器，不包含文字）
- 歌单卡片（图片，只是歌单容器，不包含歌曲名）

不要默认生成主视觉、人物、播放器、贴纸或额外装饰。只有用户补充说明明确要求时才添加。

### 组件拆分建议
- 背景层：全幅，可含装饰纹理和氛围元素
- 主视觉层：如果用户明确要求人物/核心插画，作为独立组件
- 卡片/面板层：标题卡片和歌单卡片只是文字容器，不包含用户文字
- 装饰层：独立的装饰性元素（花、星星、贴纸等）
- 文字层：所有用户可编辑的文字（标题、歌曲名、描述）

### image2 prompt 提示
- 每个组件 prompt 必须包含整体风格描述（style brief）
- 组件内需要留出文字区域的空白
- 不生成可读文字、标签、数字
- 不生成用户输入的标题或歌曲名
- 背景图片必须覆盖整张画布且不透明
- 非背景图片组件必须要求透明背景，卡片外或物体外透明
- 标题文字层大小必须贴合标题区域，不能覆盖整张画布
- 光照方向、色调在同一张海报内保持一致

### 文字层属性
- fontFamily、fontSize、fontWeight、color、textAlign、lineHeight、letterSpacing、rotation、opacity
- 多列歌单需要列坐标和列宽定义
- 长文本需要自动换行和截断规则
